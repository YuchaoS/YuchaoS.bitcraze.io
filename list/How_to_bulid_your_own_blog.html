# 如何通过vuepress+GitHub Pages搭建你的博客

这边就不介绍这两个是啥了,直接开始教程.

# GitHub Pages

# 创建仓库

username.github.io

创建完毕以后clone到本地

git clone https://github.com/<username>/<username>.github.io.git

# 测试页面

然后进入该目录创建一个测试的html文件

echo 'test' > test.html
git add .
git commit -m'test'
git push

接下来你就可以在浏览器输入<username>.github.io来访问你自己的页面了

# Vuepress

# 安装node

https://nodejs.org/en/

选择合适的版本进行下载,npm也会默认一起安装,建议不要先不要换成淘宝源,不然会导致部分包无法成功下载

下载完成后在你的terminal(cmd)里输入

npm -v

如果能看到版本号那么就ok了,不然还得加入到环境中

# 安装vuepress

npm install -g vuepress

表示全局安装vuepress,vuepress依赖许多包,所以要耐心等待下

# 构建项目

# 使用默认主题

cd <PATH> 
npm init -y

<PATH> 为你之前创建的git本地仓库目录

并构建如下目录: 默认的主题配置可按照官方样例进行书写,

https://www.vuepress.cn/theme/default-theme-config.html#%E9%A6%96%E9%A1%B5

其中的图片需要自己添加到public 目录下 然后在项目根目录下的配置文件package.json中添加:

  "scripts": {
    "docs:dev": "vuepress dev docs",
    "docs:build": "vuepress build docs",
  }

当然你得把构建好的blog push到git远程分支,所以编写一个sh脚本 touch deply.sh

#!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run docs:build

# 进入生成的文件夹
cd docs/.vuepress/dist

# 如果是发布到自定义域名
# echo 'www.example.com' > CNAME

git init
git add -A
git commit -m 'deploy'
git push -f https://github.com/<username>/<username>.github.io.git master
cd -

然后再加入package.json

  "scripts": {
    "docs:dev": "vuepress dev docs",
    "docs:build": "vuepress build docs",
    "deply":"bash deply.sh"
  },

然后你只要在terminal中输入 npm run deply 就ok了
